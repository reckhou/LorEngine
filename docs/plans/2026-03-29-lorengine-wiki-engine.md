# Lorengine Wiki Engine — Implementation Plan

**Goal:** Build the complete Lorengine wiki engine: build script, config, example doc, static site templates (HTML/CSS/JS), GitHub Actions, and supporting files.
**Architecture:** `build_wiki.py` reads `config.yml` + walks `docs/*.md` → produces `search_index.json` + copies/injects wiki templates into `_site/`. The wiki UI is vanilla HTML/CSS/JS that loads the search index client-side via Lunr.js, renders markdown via marked.js, and has an AI sidebar that calls the Claude API. All state flows through the search index JSON — the build script produces it, the frontend consumes it.
**Tech Stack:** Python 3.11+ (stdlib only), vanilla JS (ES2020+), CSS custom properties, CDN libs (Lunr.js, marked.js, highlight.js)

### Task 1: Build script + config + example doc + .gitignore

**Files:** `build_wiki.py`, `config.yml`, `docs/example-doc.md`, `.gitignore`

This is the foundation — everything else depends on the build script working.

#### config.yml

Default upstream config with sensible defaults:

```yaml
wiki:
  title: "My Wiki"
  description: "A Lorengine-powered wiki"
  accent_color: "#7F77DD"

ai:
  model: "claude-sonnet-4-6"
  system_prompt_file: "CLAUDE.local.md"
  max_tokens: 1024
  enable_prompt_caching: true

search:
  max_results: 20
  excerpt_length: 160

github_pages:
  base_url: ""
```

#### docs/example-doc.md

A minimal doc demonstrating the required frontmatter format with `##` and `###` headings.

#### build_wiki.py

Stdlib-only Python 3.11+ script. Key implementation details:

1. **Config loading:** Use `yaml` — wait, stdlib only. Use a simple YAML parser or just regex-based parsing for the flat config structure. Actually, Python stdlib doesn't include YAML. Use a minimal hand-rolled parser that handles the flat nested structure in config.yml (it's only 2 levels deep with simple values).

2. **Frontmatter parsing:** Read file, split on `---` delimiters, parse the YAML-like key-value pairs between them. Keys: `title` (required), `tags` (list), `status` (string), `last-updated` (string).

3. **Doc walking:** `pathlib.Path('docs').rglob('*.md')` — skip files without `title` in frontmatter, log warning.

4. **Heading extraction:** Regex `^#{2,3}\s+(.+)$` on each line to find `##` and `###` headings.

5. **Excerpt generation:** Strip markdown syntax from body, take first 300 chars (configurable via `search.excerpt_length` but excerpt in index is always 300; the config value is for UI display).

6. **search_index.json:** Write the JSON array to repo root. Schema per CLAUDE.md spec.

7. **Site assembly (`_site/`):**
   - Create `_site/` directory
   - Copy `wiki/*.html`, `wiki/style.css`, `wiki/wiki.js` into `_site/`
   - Copy `search_index.json` into `_site/`
   - Copy `docs/` into `_site/docs/` (raw markdown files served for the viewer)
   - Inject config values into HTML files: replace template placeholders like `{{WIKI_TITLE}}`, `{{WIKI_DESCRIPTION}}`, `{{ACCENT_COLOR}}`, `{{AI_MODEL}}`, `{{AI_MAX_TOKENS}}`, `{{BASE_URL}}`
   - If `ANTHROPIC_API_KEY` env var is set, inject it as a JS constant; otherwise leave blank

### Task 2: Wiki static site (HTML/CSS/JS templates)

**Files:** `wiki/index.html`, `wiki/page.html`, `wiki/search.html`, `wiki/style.css`, `wiki/wiki.js`
**Depends on:** Task 1 (build script must exist to test the full pipeline)

#### wiki/style.css

CSS custom properties for all colors/spacing. Two themes via `prefers-color-scheme`:
- Light mode: white background, dark text
- Dark mode: dark background, light text
- Accent color via `--accent-color` (injected from config at build time, with fallback)
- Clean, readable typography for documentation
- Responsive layout (sidebar collapses on mobile)
- Status badges for draft/review/final

#### wiki/index.html

Doc index page. Template placeholders for config injection.
- Header with wiki title + description
- List of all docs grouped by status or tags
- Each entry shows: title, tags, status badge, last-updated date, excerpt
- Link to search page
- Loads `search_index.json` at runtime to build the list
- Progressive enhancement: shows "Loading..." then replaces with doc list

#### wiki/page.html

Individual doc viewer.
- Reads `?doc=docs/example-doc.md` query param
- Fetches the raw `.md` file, renders via marked.js with highlight.js for code blocks
- Table of contents generated from headings
- AI sidebar panel (collapsible):
  - Text input + send button
  - Conversation display (last 4 turns)
  - Calls Claude API with: system prompt (from CLAUDE.local.md), doc index, current page content, conversation history, user message
  - Uses prompt caching headers for system prompt + doc index
  - Displays streaming response
- Navigation: back to index, link to search

#### wiki/search.html

Full-text search page.
- Loads `search_index.json`, builds Lunr.js index
- Search input with instant results
- Results show: title, excerpt (configurable length), tags, status badge
- Click result → navigates to `page.html?doc=<id>`

#### wiki/wiki.js

Shared logic across all pages:
- `loadSearchIndex()` — fetch and cache `search_index.json`
- `buildLunrIndex(data)` — create Lunr index from search data
- `renderDocList(data, container)` — render doc cards for index page
- `renderSearch(query, index, data)` — execute search + render results
- `renderMarkdown(md)` — marked.js + highlight.js rendering
- `initAISidebar(config)` — AI sidebar: conversation state, API calls, streaming display
- `makeAPICall(messages, systemPrompt, docIndex, currentDoc)` — Claude API call with prompt caching
- Theme toggle (respects system preference, allows manual override)
- Base URL handling for GitHub Pages subdirectory serving
